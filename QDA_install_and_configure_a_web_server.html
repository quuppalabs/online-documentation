Install and configure a web server

The essential steps of installing and configuring a simple QDA server are very similar to QPE server installation with a few differences. These steps are described below.

  1. Install the latest Java:

    Download and install 64-bit Java SE Runtime Environment. Download JDK, Server JRE or JRE version of Java based on your needs. For example, by follwing the steps in OpenJDK web page.

    Important:
    If you are running 64-bit operating system, make sure you are installing the 64-bit version of Java as the 32-bit version cannot allocate more than 2 GB of memory, which might be needed in some cases.
  2. Install and configure Apache Tomcat:

    Below you can find instructions to install Apache Tomcat which is our preferred Java web server. If you are more familiar with some other web server platform and you wish to run that instead of Tomcat, you can do that, but then you are on your own.

    Here we install Tomcat 8.5.x in Ubuntu.
    • Run on terminal: sudo apt-get install tomcat8
    • Create a file ‘setenv.sh’ under the directory “/usr/share/tomcat8/bin/” by running in terminal:

      sudo nano /usr/share/tomcat8/bin/setenv.sh

    • Add the following lines to the file, save and exit:

      export JAVA_OPTS="-Xmx4096m -XX:+UseConcMarkSweepGC

      -Djava.awt.headless=true -DaggregatorFolder=<Aggregator folder path>"

    • The -Xmx option sets the maximum heap memory size for Java virtual machine, set

      here set to 4096 MB (4 GB). If you want to increase the size, you can do it here. The

      -DaggregatorFolder option defines the folder where Aggregator project is saved.

    • Refer to the Tomcat documentation for more information. Contact your IT administrator for further assistance.

  3. Deploy QDA

    Download the latest Quuppa Data Aggregator war file ‘qda.zip’ from the Quuppa Customer Portal and extract the files.

    • Copy the file qda.war to Tomcat’s webapps directory /var/lib/tomcat8/webapps/
      Important:
      If you are doing a system update, before copying the new qda.war file, remove both the previous .war file and related directory (having the same name as of the .war file) from the “…./tomcat8/webapps/” directory.

      This is to ensure that the Tomcat will run 1 (and only 1) qda.war instance at the time. Not removing previous versions of the QDA application running under Apache Tomcat will have an impact on your system stability!

    • Restart Tomcat.
    • Go to xxx.xxx.xxx.xxx:8080/qda/ to see the Quuppa Data Aggregator Web Console.
