Edit Project Defaults

The project defaults (e.g. operation mode, tracking area type, zone group, Locator type, locator sensitivity and Locator type) are set when you create a new project in the QSP using the New Project Wizard. These are the default settings for new object added to the project. However, if you notice that you need to change these settings, it is possible through the general project settings. Please follow the instructions below to change these settings.
Note: These settings affect Quuppa Positioning Engine (QPE) assumptions and calculations.
  1. Open the project in the QSP.
  2. In the object tree on the left, select the project file.

  3. In the panel on the right, check that you are in the General tab and find the Project defaults section.

  4. Edit the Project defaults as needed:

    1. Operation mode: Select Proprietary or BLE.
    2. Tracking Area type: Select Open, SemiConfined or Confined.
    3. Zone Group: Select Default or other created Zone groups.
    4. Locator Type: Select Q17 or Q35.
    5. Locator sensitivity: Select the appropriate sensitivity level.
    6. Locator height: Select the default installation height for Locators in meters.
    7. Unknown Tags use this Tag Group's settings: Select which Tag Group's settings should be used for ungrouped tags (i.e. tag ID not saved to the project file). Note that ungrouped tags will not be assigned to this group, it's only using the settings of this tag group. Read more about tag groups in the Tag Groups section and about managing tags in the Manage Tags section.
      Note: Remember that these are the default settings for new objects added to the project file. Changing them will not affect already existing objects in the project file.

      The settings for individual or groups of object can also be changed manually (e.g. if you have Q35 Locators in most of the deployment area but Q17 Locators in a specific area of your deployment).
