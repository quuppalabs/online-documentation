Create Location-Based Temporary Commands

Zone Groups can be used to trigger location-based temporary commands for tags. This means that the tags behave in a certain way when they enter the defined zones and revert back to normal behaviour when they exit the zone. To create new and custom location-based command, follow the steps below.
  1. Open the project in the QSP.
  2. In the menu bar at the top, open the Deployment tab and select the Temporary command editor.

  3. In the Temporary command editor window that opens, click the New temporary command button to create a new temporary command.

  4. The QSP will prompt you to give the new temporary command an ID (i.e. a name so that you can identify it from temporary command lists as needed).

  5. In the panel on the right, edit the properties as needed and close the window. The changes are saved automatically.

  6. Enable the new location-based temporary command for the relevant tag groups. For more instructions on how to do so, see the Enable Location-Based Temporary Commands section.
  7. Save and submit the project for the changed to take effect.

    From now on, during tracking, whenever a tag enters a zone with a defined temporary command, the tag will start behaving according to the defined temporary state behaviour.

    Note:
    The zone-based location information can be made available through the API. For more information, see the Quuppa API Documentation.
