Edit Tag Configurations

Before you can track your tags, you will need select the desired configurations, i.e. behaviours, for the tags in your project. This is done with the tag configuration editor.

With the editor, you can choose from the available default tag configurations, edit the behaviours of tags in different states as needed, as well as create custom tag profiles for different types of tags used in your project.

In most cases, we recommend using the available default tag configurations. They have been optimised, based on Quuppa's experience in the field over the years, to provide the best performance for the defined uses:

  • ASSET_TAG - for tracking assets, e.g. pallets or products.
  • ID_BADGE - for locating people, e.g. wearing wearable tags or card tags on a lanyard.
  • DEMO_TAG - for providing smooth and real-time demonstrations and test results. Please note that this configuration consumes more tag battery power than ASSET_TAG or ID_BADGE configurations and so is not typically recommended for production systems.
  • IDLE - for conserving battery power by placing the tag into power saving mode quickly.

However, if the default tag configurations don't work for your use case, they can also be customised as needed.

This section will walk you through using the editor to view current settings, edit the storage, default and triggered state settings and creating new tag profiles.
